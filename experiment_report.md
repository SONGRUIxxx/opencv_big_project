## 摘要

本实验围绕"鱼眼域直接处理"（Route A）与"校正后处理"（Route B）两条技术路线，实现了帧差法、Farneback 稠密光流和 Lucas-Kanade 稀疏光流三种传统运动提取算法，配合连通域分析与光流中值传播完成短时目标跟踪。在 WoodScape 数据集 130 个样本上，以像素级 IoU、Precision、Recall、F1-score 为指标进行量化评估，并对比了图像中心区域与边缘区域的性能差异。实验结果表明：校正后处理（Route B）在全图 IoU 和 F1 上均有一定提升，其中边缘区域提升幅度最大；Farneback 光流具有最高的召回率（~0.80），但精确率较低；帧差法的精确率相对较高但召回率不足。综合来看，鱼眼校正能够部分缓解边缘畸变对运动分析的影响，但受限于 2 帧场景，传统方法的绝对指标仍然偏低。

## 1. 问题背景与理论基础

### 1.1 透视相机 vs 鱼眼相机

传统的**透视相机**遵循小孔成像模型：三维空间中的直线在图像中仍然保持为直线。其视场角（FOV）通常限制在 60°~90° 范围内，因为当 FOV 超过 90° 时，透视投影会导致图像边缘的物体被极度拉伸。

**鱼眼相机**通过引入显著的径向畸变来换取超大视场角（可达 190° 甚至更大）。其成像不遵循小孔模型，而是遵循特定的**投影函数**，将入射角 θ（光线与光轴的夹角）映射为像面上的径向距离 r：

$$r = f(\theta)$$

不同的投影函数对应不同种类的鱼眼镜头：

| 模型 | 投影函数 | 特点 |
|------|---------|------|
| 等距投影 | r = f·θ | 角分辨率均匀 |
| 等立体角投影 | r = 2f·sin(θ/2) | 面积比例保持 |
| 正交投影 | r = f·sin(θ) | 边缘压缩最强 |
| 体视投影 | r = 2f·tan(θ/2) | 类似透视 |

### 1.2 WoodScape radial_poly 模型

本实验使用的 WoodScape 数据集采用**径向多项式模型**（radial_poly），其投影函数为：

$$r(\theta) = k_1\theta + k_2\theta^2 + k_3\theta^3 + k_4\theta^4$$

其中：
- **θ**：入射光线与光轴之间的夹角（弧度），范围 [0, θ_max]
- **r(θ)**：像点到图像主点的距离（像素）
- **k₁**：一阶系数，近似于等效焦距（本数据 k₁ = 339.749）。在 θ 较小时（图像中心），r ≈ k₁·θ，近似等距投影
- **k₂, k₃, k₄**：高阶修正项，用于拟合真实镜头的非理想特性。k₂ = -31.988 为负值，意味着在大 θ 处（边缘）径向距离的增长速度减缓（桶形畸变特征）

**该模型与 OpenCV 标准鱼眼模型的本质区别**：

OpenCV 的 `cv::fisheye` 模型遵循：
$$\theta_d = \theta \cdot (1 + k_1\theta^2 + k_2\theta^4 + k_3\theta^6 + k_4\theta^8)$$

其中 θ_d 是畸变后的入射角。这是**奇次多项式**模型，而 WoodScape 的 radial_poly 是**全次多项式**模型（包含奇次和偶次项，从 1 次到 4 次）。两个模型的数学形式不同，**无法直接互换参数**。因此，本实验不能简单调用 OpenCV 的 `cv::fisheye::initUndistortRectifyMap`，而必须自行实现校正映射。标定 JSON 的解析采用内联最小解析器，无需第三方 JSON 库，确保工程零外部依赖、开箱即编。

图像主点坐标由标定参数中的偏移量计算：
$$c_x = \frac{W}{2} + \text{cx\_offset} = 640 + 3.942 = 643.942$$
$$c_y = \frac{H}{2} + \text{cy\_offset} = 483 + (-3.093) = 479.907$$

### 1.3 鱼眼畸变对运动分析的影响

鱼眼镜头的径向畸变给运动提取和目标跟踪带来三个核心挑战：

**（1）几何尺度不一致**。同一物理尺寸的物体，在图像中心占据的像素面积远大于在图像边缘。一个 2 米宽的车辆，在图像中心可能占 200 像素，在边缘仅占 80 像素。这意味着固定大小的形态学操作核在中心和边缘的实际物理覆盖范围差异巨大。

**（2）运动轨迹弯曲**。在透视图像中，沿直线匀速运动的物体产生直线的像素轨迹。但在鱼眼图像中，同一物理运动在边缘区域呈现明显的弧线轨迹，因为径向畸变改变了局部的像素/物理距离映射关系。

**（3）光流方向畸变**。Farneback 等光流算法假设局部灰度在平移下保持不变。但鱼眼畸变意味着物体的表观形状随位置变化，破坏了这一假设。尤其在图像边缘，同一刚体运动在不同位置产生不同方向和大小的光流向量。

这些挑战引出了本实验的核心问题：**是先校正畸变再做运动分析更好（Route B），还是直接在鱼眼域处理（Route A）？**

数据来源于 **WoodScape** 公开自动驾驶鱼眼数据集，选取**前视相机（Front View, FV）**视角。WoodScape 是首个包含语义标注的大规模鱼眼自动驾驶数据集，本实验使用其运动标注子集。

### 2.2 数据结构

```
homework2/
├── rgb_images/{ID}_FV.png              # 当前帧（鱼眼 RGB, 1280×966）
```

真值掩膜为单通道 PNG 图像（1280×966, 8-bit 灰度），每个像素值为：

## 3. 算法设计

### 3.1 总体框架

本实验实现两条独立的技术路线，使用相同的运动提取和跟踪核心算法，仅在输入图像的坐标域上有所不同：

```
                    ┌─────────────────┐
    当前帧 + 前一帧  │   Route A        │
    (鱼眼域)  ──────▶│ 鱼眼域运动提取   │──▶ 运动掩膜 + 跟踪 + 评估
                    │ 帧差/Farneback/LK │
                    └─────────────────┘

                    ┌─────────────────┐     ┌──────────────────┐
    当前帧 + 前一帧  │   Undistorter   │     │   Route B        │
    (鱼眼域)  ──────▶│ 鱼眼→透视校正   │────▶│ 透视域运动提取   │──▶ 运动掩膜 + 跟踪 + 评估
                    │ (预计算remap表) │     │ 帧差/Farneback/LK │
                    └─────────────────┘     └──────────────────┘
```

**Route A 与 Route B 的本质区别**：不是算法不同，而是**分析的坐标系不同**。Route A 在鱼眼的径向距离-方位角坐标下工作，Route B 在透视的笛卡尔坐标下工作。评估时，Route A 的预测掩膜与原始真值掩膜直接比较；Route B 的预测掩膜与**校正后的真值掩膜**比较（校正用最近邻插值保持二值性质）。

### 3.2 鱼眼校正算法（Undistorter）

#### 3.2.1 为什么不能直接调用 OpenCV

OpenCV 的 `cv::fisheye::initUndistortRectifyMap` 基于奇次多项式模型（θ_d = θ(1+k₁θ²+k₂θ⁴+...)），而 WoodScape 的 radial_poly 是全次多项式（r = k₁θ+k₂θ²+k₃θ³+k₄θ⁴）。两个模型的参数空间不同，无法通过参数换算来复用。

#### 3.2.2 校正原理（反向映射法）

校正采用的是**反向映射**策略：遍历输出图像（透视）的每个像素，计算它在输入图像（鱼眼）中的对应位置。

**Step 1：确定目标透视相机参数**

选择输出视场角 HFOV = 120°（约 2.09 rad），输出宽度 = 1024 像素。这个选择的依据：
- 120° 在鱼眼有效视野（~150°）内，不会产生大量无效像素
- 透视畸变可控（边缘拉伸不严重）
- 1024 宽度在细节保留和计算效率之间平衡

计算焦距和输出高度：
$$f = \frac{W_{out}}{2 \cdot \tan(\text{HFOV}/2)} = \frac{1024}{2 \cdot \tan(60°)} = \frac{1024}{2 \cdot 1.732} \approx 295.6 \text{ px}$$

输出高度由原始图像的宽高比自动推导：
$$\text{VFOV} = 2 \cdot \arctan\left(\tan(\text{HFOV}/2) \cdot \frac{H_{fish}}{W_{fish}}\right)$$
$$H_{out} = 2 \cdot f \cdot \tan(\text{VFOV}/2) \approx 773 \text{ px}$$

**Step 2：对每个输出像素 (u, v) 执行反向映射**

对于透视图像上的像素 (u, v)：

1. **归一化坐标**：将像素坐标转换为以主点为中心的归一化坐标
   $$n_x = \frac{u - c_x^{out}}{f}, \quad n_y = \frac{v - c_y^{out}}{f}$$

2. **求入射角 θ 和方位角 φ**：
   $$\theta = \arctan\left(\sqrt{n_x^2 + n_y^2}\right)$$
   $$\phi = \text{atan2}(n_y, n_x)$$

   这里利用了透视相机的基本关系 r_norm = tan(θ)。

3. **正向投影求鱼眼像面距离**：将 θ 代入 WoodScape radial_poly 模型
   $$r_{fish} = k_1\theta + k_2\theta^2 + k_3\theta^3 + k_4\theta^4$$

   使用 Horner 法计算以提高数值精度：
   $$r_{fish} = (((k_4\theta + k_3)\theta + k_2)\theta + k_1)\theta$$

4. **映射到鱼眼像素坐标**：
   $$u_{fish} = c_x^{fish} + r_{fish} \cdot \cos(\phi)$$
   $$v_{fish} = c_y^{fish} + r_{fish} \cdot \sin(\phi)$$

5. **有效性检查**：若 θ > θ_max（鱼眼最大入射角，约 1.97 rad / 113°），则标记为无效（映射值 = -1），`cv::remap` 会用黑色填充。

**Step 3：存储为 remap 查找表**

所有 1024×773 个像素的 (u_fish, v_fish) 映射预先计算并存入 `cv::Mat`（CV_32FC1），运行时通过一次 `cv::remap` 调用即可完成校正。由于所有 130 个样本共享相同的标定参数，remap 表仅构建一次，全局复用。

#### 3.2.3 真值掩膜的校正

真值掩膜在 Route B 中也必须校正到透视域，否则坐标空间不匹配无法评估。使用 `cv::INTER_NEAREST`（最近邻插值）而非默认的线性插值，原因是：线性插值会在掩膜边缘产生 0 到 255 之间的灰度过渡值，破坏二值性。最近邻插值保证校正后的掩膜仍然只包含 0 和 255 两个值。

#### 3.2.4 牛顿法求解反向投影

在计算 max_θ（鱼眼最大入射角）时需要用到反向投影——已知像面距离 r，求入射角 θ：

$$r(\theta) - r_{target} = 0$$

由于 r(θ) 对鱼眼镜头是单调递增的（θ↑ → r↑），Newton-Raphson 迭代可快速收敛：

$$\theta_{k+1} = \theta_k - \frac{r(\theta_k) - r_{target}}{r'(\theta_k)}$$

其中导数：
$$r'(\theta) = k_1 + 2k_2\theta + 3k_3\theta^2 + 4k_4\theta^3$$

初始猜测 θ₀ = r/k₁（线性近似），收敛容忍度 1e-8，通常 5~10 次迭代即可收敛。

### 3.3 运动提取算法

#### 3.3.1 帧差法

**核心思想**：如果场景中没有运动物体，前后两帧的像素值应几乎不变（仅受传感器噪声影响）。当像素值发生显著变化时，该位置大概率存在运动物体。

**完整流程与参数推导**：

```
灰度图(gray_prev) + 灰度图(gray_curr)
  │
  ├─ 1. 高斯模糊 (kernel=5)
  │   为什么需要模糊？
  │   传感器噪声（~2-5 灰度级）会在帧差中产生孤立噪点。
  │   5x5 高斯核平滑后，单个像素的随机噪声被抑制到 <1 灰度级，
  │   而真实运动边界（跨越多像素的梯度）被保留。
  │   为什么是 5 而非 3？5 对 1280×966 图像提供了约 0.4% 的空间平滑，
  │   足以抑制噪声而不显著模糊物体轮廓。
  │
  ├─ 2. 绝对值差分 cv::absdiff()
  │   为什么用 absdiff 而非直接相减？
  │   CV_8UC1 是无符号类型，(150-200) 会溢出为 206（而非 -50）。
  │   absdiff 正确处理了这种情况。
  │
  ├─ 3. 固定阈值化 (threshold=25)
  │   为什么是 25 而非自适应阈值（如 Otsu）？
  │   a) 两帧场景下，运动像素通常占 <20%，Otsu 假设的双峰分布不成立
  │   b) 传感器噪声 ~2-5 灰度级，光照轻微变化 ~5-10 灰度级
  │      真实运动导致的差异通常 >30 灰度级
  │   c) 25 是一个保守的折中：大于噪声和光照波动的上限（~15），
  │      同时不会过于严格而漏掉低对比度运动（如阴影中的行人）
  │
  ├─ 4. 形态学开运算 (椭圆核 3×3)
  │   为什么需要开运算？
  │   阈值化后残留的孤立白点（单像素噪声）面积 < 9 像素，
  │   开运算（先腐蚀后膨胀）能移除这些噪点而不改变较大的运动区域。
  │   为什么用椭圆核而非矩形核？
  │   真实运动区域在图像中的轮廓是任意方向的，
  │   椭圆核没有方向偏好，对圆形和线形区域同样友好。
  │
  └─ 5. 形态学闭运算 (椭圆核 5×5)
      为什么需要闭运算？
      运动物体内部可能存在纹理均匀的区域（如车门、路面），
      这些区域在帧差中表现为空洞。闭运算（先膨胀后腐蚀）
      填充这些空洞，使运动区域更连续。
      为什么闭运算核(5)比开运算核(3)大？
      需要填充的空洞通常比需要去除的噪点大。
```

**输出**：二值掩膜（255=运动, 0=静止） + 运动像素占比。

#### 3.3.2 Farneback 稠密光流

**核心思想**：用二次多项式对每个像素的局部邻域进行灰度曲面拟合。如果两帧之间该邻域发生了平移，那么两个多项式的系数之间存在确定性的关系，由此可以解出位移向量。

**数学模型**（简化）：

对邻域内的灰度分布，用二次多项式逼近：
$$f(x) \approx x^T A x + b^T x + c$$

其中 A 是 2×2 对称矩阵（描述曲率），b 是 2D 向量（描述梯度），c 是标量（描述亮度）。

若第一帧在位置 x 处的多项式为 f₁(x)，第二帧在 x+d 处有相同的多项式 f₂(x+d) = f₁(x)，则：
$$f_2(x) = f_1(x - d) = (x-d)^T A_1 (x-d) + b_1^T(x-d) + c_1$$

展开并比较系数，可得位移 d 的闭式解（需要 A₁ 可逆）：
$$d = -\frac{1}{2}A_1^{-1}(b_2 - b_1)$$

**为什么需要稠密光流而非仅用帧差法？**

帧差法只能回答"动了没有"（二值），Farneback 能回答"往哪动、动多快"（向量场）。这对于后续跟踪至关重要：知道运动方向后，可以预判目标在下一帧的位置。

**关键参数解释**：

| 参数 | 值 | 为什么这样选 |
|------|-----|-------------|
| `pyr_scale` | 0.5 | 金字塔每层缩小一半。需要金字塔的原因：大物体的位移可能超过局部窗口的大小，在粗层（低分辨率）上物体会落入窗口内 |
| `levels` | 3 | 3 层金字塔 × 2³=8 倍位移范围。原始 1280×966 上 15×15 窗口在顶层对应 120×120 的有效覆盖 |
| `winsize` | 15 | 局部窗口大小。15 在"空间精度"和"抗孔径效应"之间平衡——太小则遇到均匀区域无解，太大则模糊了运动边界 |
| `poly_n` | 5 | 5 阶多项式有足够自由度逼近典型的灰度变化（平滑的斜坡、曲面），同时不因阶数过高而对噪声过拟合 |
| `iterations` | 3 | 3 次迭代足够消除位移估计的偏差。更多迭代收益递减 |
| 幅值阈值 | 2.0 px | 位移 <2 像素的光流被认为是传感器噪声或缓慢的光照漂移，不计入运动掩膜 |

**输出**：稠密光流场（每个像素有 vx, vy）+ 阈值化运动掩膜。

#### 3.3.3 Lucas-Kanade 稀疏光流

**核心思想**：Farneback 试图为**每个像素**估计运动（稠密），Lucas-Kanade 只为**容易跟踪的特征点**估计运动（稀疏）。它基于三个假设：

1. **亮度恒定**：特征点在两帧之间的灰度不变
2. **小位移**：特征点的运动在局部窗口内近似为平移
3. **空间一致性**：窗口内所有像素共享相同的运动向量

在角点 p 的邻域窗口内，求解光流方程的超定线性系统：
$$\begin{bmatrix} I_x(p_1) & I_y(p_1) \\ I_x(p_2) & I_y(p_2) \\ \vdots & \vdots \end{bmatrix} \begin{bmatrix} v_x \\ v_y \end{bmatrix} = -\begin{bmatrix} I_t(p_1) \\ I_t(p_2) \\ \vdots \end{bmatrix}$$

其中 I_x, I_y 是空间梯度，I_t 是时间梯度（帧间差）。用最小二乘法求解。

**与 Farneback 的定性区别**：

| 维度 | Farneback | Lucas-Kanade |
|------|-----------|--------------|
| 密度 | 每个像素都算 | 只在角点处算 |
| 速度 | 慢（O(N²)） | 快（O(kN)，k 是角点数） |
| 精度 | 在纹理区域好，在边缘处有孔径问题 | 在角点处精度高 |
| 适用 | 需要完整运动场的场景 | 需要可靠特征跟踪的场景 |
| 盲区 | 无（所有像素都有输出） | 均匀区域无角点→无输出 |

**关键参数解释**：

| 参数 | 值 | 为什么这样选 |
|------|-----|-------------|
| `max_corners` | 500 | 在 1280×966 图像上，500 个角点平均间距约 52 像素，能覆盖大部分运动区域而不冗余 |
| `quality_level` | 0.01 | 为最大角点响应的 1%。宽松设置确保低纹理道路场景也能检测到足够角点 |
| `min_distance` | 10 | 防止角点扎堆在同一纹理丰富区域（如车道线边缘），保证空间分布均匀 |
| `win_size` | 21 | 21×21 窗口提供 441 个方程，求解 2 个未知数，冗余度高，数值稳定 |
| `max_level` | 3 | 与 Farneback 同理，3 层金字塔处理大位移 |
| `min_displacement` | 2.0 | 过滤位移 <2 px 的"静止"点（实际可能是传感器噪声或树叶微动） |

**输出**：前后帧匹配的角点对（带状态标记） + 位移向量。

### 3.4 目标跟踪

#### 3.4.1 连通域分析

从二值运动掩膜中提取独立的运动区域：

1. `cv::connectedComponentsWithStats` 以 **8-邻接**做连通域标记
   - 为什么 8-邻接而非 4-邻接：运动区域的对角方向邻接在实际中是连续的，4-邻接会错误地把一个运动物体拆成多个碎片
2. 过滤条件：
   - **min_area = 100 px**：排除 <100 px 的噪声碎片（单个噪点 + 形态学残余通常 <50 px）
   - **max_area_ratio = 0.5**：排除异常大的"运动"（如全图曝光变化导致整帧被标记）
3. 对每个有效区域提取：边界框、质心、面积

#### 3.4.2 光流中值传播

利用 Farneback 稠密光流对连通域结果进行修正：

1. 对每个连通域，收集其边界框内所有光流向量
2. 取**中值**（而非均值）作为该区域的整体运动速度
   - 为什么用中值：区域边缘处的光流可能包含背景侵入的异常值，中值对这些异常鲁棒（breakdown point = 50%）
3. 质心和边界框按中值向量平移

#### 3.4.3 光流中值传播

利用 Farneback 稠密光流对连通域结果进行速度估计：

1. 对每个连通域，收集其边界框内所有光流向量
2. 取**中值**（而非均值）作为该区域的整体运动速度
   - 为什么用中值：区域边缘处的光流可能包含背景侵入的异常值，中值对这些异常鲁棒（breakdown point = 50%）
3. 将中值速度向量存储为区域的 velocity，用于可视化和后续跟踪

**为什么不在两帧场景下使用 Kalman 滤波**：Kalman 滤波在两帧退化情形下退化为简单的加权平均（帧 1 初始化→帧 2 预测+更新），不能提供有意义的轨迹预测。对于独立的相邻帧对（不同 ID 之间不属于同一视频序列），每个样本构成独立的运动分析单元，因此采用更直接的连通域分析 + 光流速度估计策略。

### 3.5 评估指标

| 指标 | 公式 | 物理含义 | 解读要点 |
|------|------|---------|---------|
| **IoU** | TP/(TP+FP+FN) | 预测与真值的重叠度 | 1.0=完全重叠，0.0=完全不沾。FP 和 FN 同时惩罚 |
| **Precision** | TP/(TP+FP) | 预测的"纯度" | 高 P = 很少把静止背景标为运动。但把所有像素标为静止也能 P=1 |
| **Recall** | TP/(TP+FN) | 真值的"覆盖率" | 高 R = 很少漏掉运动区域。但把所有像素标为运动也能 R=1 |
| **F1-score** | 2PR/(P+R) | P 和 R 的调和平均 | 综合指标。只有 P 和 R 都高时 F1 才高。比算术平均更严格 |

**为什么用调和平均（F1）而非算术平均？**

调和平均对低值更敏感。如果 Precision=0.9 但 Recall=0.1，算术平均=0.5（看似还行），调和平均=0.18（暴露了 Recall 的问题）。这更符合评估直觉：算法必须同时做好"不错报"和"不漏检"。

### 3.6 中心/边缘区域划分

**划分原理**：鱼眼畸变是径向对称的——离光心越远，畸变越严重。因此按**像素到光心的欧氏距离**划分区域：

- **中心区域**：距离 < 最大距离的 40%。畸变可近似为线性，与透视相机差异最小
- **边缘区域**：距离 > 最大距离的 60%。畸变非线性显著，对运动分析挑战最大
- **中间带**（40%~60%）：不计入任一类。保证中心/边缘有明显区分，避免边界模糊

中心/边缘分别评估后，对比两区域的 IoU 差异即可量化鱼眼畸变对算法的影响程度。

## 5. 实验结果

### 5.1 校正效果可视化

选取代表性样本，观察鱼眼原图与透视校正图的差异。

**输出文件**：`output/visualizations/{ID}/{ID}_undistorted_comp.png`

**读图指南**：
- **左半部分（原始鱼眼图像）**：注意图像边缘的车道线、路沿呈现明显的弧形弯曲，这是鱼眼镜头的桶形畸变特征。靠近图像中心的物体（如前方车辆）畸变较小，形状接近真实。
- **右半部分（透视校正图像）**：弯曲的车道线和路沿被"拉直"，恢复了透视投影中直线保持直线的特性。注意图像四个角出现了黑色区域——这些超出了鱼眼有效视野范围（HFOV 120° vs 鱼眼有效 ~150°），在 remap 表中被标记为无效。
- **图像尺寸变化**：鱼眼原图 1280×966，校正后 1024×773。缩小不是因为信息丢失，而是因为 120° HFOV 的透视投影覆盖的像素面积自然比 150° 鱼眼投影小。

**关键观察**：校正后，车辆、行人等目标在图像不同位置的外观尺度更加一致（透视投影的尺度变化仅与深度有关，与图像位置无关），这为后续的运动分析提供了更好的几何基础。

### 5.2 运动检测可视化

#### 5.2.1 帧差法

**输出文件**：`output/visualizations/{ID}/{ID}_route_A_frame_diff.png` 和 `{ID}_route_B_frame_diff.png`

**读图指南**：
- **绿色半透明区域**：算法判定为运动像素的位置
- **绿色越密集**：运动越剧烈或运动物体越大
- **如果绿色散碎**：说明阈值设置可能偏低，少量噪声被误检
- **鱼眼边缘的绿色往往更散碎**：原因是畸变使同一物体在边缘被拉伸，相邻像素的帧差值分布更离散

#### 5.2.2 Farneback 稠密光流

**输出文件**：`output/visualizations/{ID}/{ID}_route_A_farneback_flow.png` 和 `{ID}_route_B_farneback_flow.png`

**读图指南（重点）**：
- **色相（Hue，颜色）→ 运动方向**：
  - 🔴 **红色/橙色**：向右运动
  - 🔵 **蓝色/青色**：向左运动
  - 🟡 **黄色**：向下运动
  - 🟢 **绿色**：向上运动
  - 中间色调对应斜向运动
- **明度（Value，亮度）→ 运动速度**：
  - 越**明亮**：运动越快（光流幅值大）
  - 越**暗淡**：运动越慢
  - **纯黑色**：静止区域（幅值 < 2.0 px 阈值）
- **右下角色轮图例**：一个圆形色轮，外圈标有方向对应的颜色。使用时可以对比光流图中的颜色和色轮图例，确定运动方向

**鱼眼域 vs 透视域的典型差异**：
- 在鱼眼域中（Route A），同一辆向前行驶的车，在图像中心和边缘的光流方向可能不同（中心向右，边缘因畸变偏向右上方）
- 在透视域中（Route B），同一物体的光流方向更加一致

#### 5.2.3 Lucas-Kanade 稀疏光流

**输出文件**：`output/visualizations/{ID}/{ID}_route_A_lk_sparse.png` 和 `{ID}_route_B_lk_sparse.png`

**读图指南**：
- 🟢 **绿色圆点**：前一帧中检测到的角点（特征点）
- 🟡 **黄色箭头**：该角点的运动向量（从绿点指向当前帧的对应位置）
  - 箭头方向 = 运动方向
  - 箭头长度 = 位移大小（在 2 帧之间）
- **点的分布特征**：
  - 角点集中在**高纹理区域**（车道线边缘、车辆轮廓、路面标记）
  - **均匀路面、天空**等处几乎无点（这些区域没有可跟踪的特征）
- **箭头的一致性**：同一运动物体上的箭头通常方向一致——这验证了该物体作为刚体运动的假设

### 5.3 跟踪可视化

**输出文件**：`output/visualizations/{ID}/{ID}_route_A_tracking.png` 和 `{ID}_route_B_tracking.png`

**读图指南**：
- 🟢 **绿色矩形框**：跟踪到的运动区域（连通域的轴对齐边界框）
- `ID:X` **标签**：区域的唯一编号（从 0 开始）
- 🔵 **蓝色圆点**：区域的质心位置
- 🔴 **红色箭头**：该区域估计的运动速度向量
  - 方向 = 预测的运动方向
  - 长度 = 速度大小（放大了 3 倍以增强可视性）

**如何判断跟踪质量**：
- 如果红色箭头方向与肉眼判断的运动方向一致 → 跟踪良好
- 如果方框包含了大量背景区域 → 连通域提取偏大（形态学闭运算核可能过大）
- 如果同一物体被拆成多个小框 → 连通域破碎（需要更大的闭运算核或调整帧差阈值）

以下数据来自 `results_summary.csv`，对全部 130 个样本取均值：

#### 5.5.1 帧差法

| | 中心 | 0.044 | 0.058 | 0.394 | 0.076 |
| | 边缘 | 0.023 | 0.028 | 0.251 | 0.041 |

**数据解读**：

- **边缘提升幅度最大**：Route B 边缘 IoU 比 Route A 高 32.5%（0.023→0.031），印证了校正对缓解边缘畸变最有效
- **Recall 始终远高于 Precision**（0.2~0.5 vs 0.03~0.07）：帧差法倾向于"宁可多报不少报"——阈值 25 把大量微弱差异也标为运动

#### 5.5.2 Farneback 光流

| 路线 | 区域 | IoU | Precision | Recall | F1 |
|------|------|-----|-----------|--------|-----|
| **Route A**（鱼眼） | 全图 | 0.023 | 0.025 | **0.803** | 0.040 |
| | 中心 | 0.026 | 0.032 | 0.695 | 0.044 |
| | 边缘 | 0.021 | 0.021 | 0.428 | 0.036 |
| **Route B**（校正） | 全图 | **0.023** | **0.026** | 0.733 | **0.040** |
| | 中心 | 0.028 | 0.033 | 0.615 | 0.046 |
| | 边缘 | 0.022 | 0.024 | 0.353 | 0.037 |

**数据解读**：
- **Recall 极高（0.80）但 Precision 极低（0.025）**：Farneback 几乎检测到了所有运动像素（FN 很少），但同时也将大量静止像素误判为运动（FP 极多）。Recall 是帧差法的 1.7 倍，但 Precision 仅为帧差法的 62%
- **Route A 的 Recall 高于 Route B**（0.803 vs 0.733）：校正过程中线性插值可能平滑了光流场，轻微降低了边缘的灵敏度

|------|-----------|--------|-----|------|------|

| Farneback | 0.025 | **0.803** | 0.040 | 最慢 | 全面但高噪，适合需求方向信息的场景 |

  - 点落在对角线**下方（右下）** → 边缘效果比中心差（绝大多数样本应在此）
  - 点落在对角线上方 → 该样本边缘恰好有简单/明显的运动模式
- **解读**：绝大多数绿点聚集在对角线下方，定量证明了鱼眼畸变导致边缘区域运动检测性能下降

---

## 6. 分析与讨论

### 6.1 Route A vs Route B：为什么校正有效

从帧差法的数据看，Route B 在边缘区域将 IoU 从 0.023 提升到 0.031（+32.5%）。其原因可以从几何层面解释：

在鱼眼域中（Route A），图像边缘的物体被径向压缩——同一辆 2 米宽的汽车，在边缘仅占 50 像素宽，在中心占 120 像素宽。形态学操作的固定核（如 5×5 像素）在中心和边缘覆盖的物理面积不同：中心覆盖小区域（约 0.5m²），边缘覆盖大区域（约 2m²）。这导致边缘的形态学处理要么"过度"（核相对太大，模糊了运动边界），要么"不足"（如果用更小的核则中心效果下降）。

在校正域中（Route B），透视投影的尺度一致性意味着同一物体在图像各处占据相近的像素面积，形态学核的物理意义统一。这是 Route B 优于 Route A 的根本原因。

**但校正也有代价**：校正图像的四个角存在黑色无效区域（超出鱼眼 FOV 的部分），且校正过程的高阶插值（INTER_LINEAR）会轻微模糊图像，这对 Farneback 的梯度计算可能有细微影响（解释了为什么 Farneback 的 Recall 在 Route B 反而略低）。

### 6.2 中心 vs 边缘差异

中心区域的 F1 约为边缘的 1.7~1.9 倍。即使在校正后（Route B），边缘性能仍然显著低于中心。这说明校正**缓解但未消除**边缘劣势。原因包括：

1. **校正本身的信息损失**：边缘区域的像素在校正过程中被拉伸放大，相当于对低分辨率区域做了上采样，信息量天然不足
2. **鱼眼边缘的光学质量下降**：真实镜头的边缘存在色差、彗差和渐晕，这些在校正中无法恢复
3. **运动物体更可能在图像近处（中心）**：自动驾驶场景中，大多数运动物体（车辆、行人）在自车前方，自然集中在图像下半部分的中心区域；边缘是天空、建筑等静止背景

### 6.3 三种运动检测方法对比

**帧差法**：
- ✅ 速度最快（仅需 absdiff + 阈值 + 形态学），适合实时系统
- ✅ Precision 最高（0.040），误检相对少
- ❌ Recall 不足（0.474），漏掉了一半以上的运动区域
- ❌ 无法提供运动方向信息

**Farneback 稠密光流**：
- ✅ Recall 极高（0.803），几乎捕获了所有运动
- ✅ 提供了完整的 2D 运动场（方向 + 速度），信息量丰富
- ❌ Precision 最低（0.025），大量静止像素被误报
- ❌ 计算量最大，不适合实时应用

**Lucas-Kanade 稀疏光流**：
- ✅ 角点级精度高，适合特征级运动分析
- ✅ 计算量介于两者之间
- ❌ **无法直接生成运动掩膜**（仅输出特征点对），因此未纳入像素级评估
- ❌ 在无纹理区域（均匀路面、天空）无输出

**三者互补关系**：帧差法提供粗略但可靠的运动区域（高 P），Farneback 提供完整但噪声大的运动场（高 R），Lucas-Kanade 在角点上提供高精度的位移测量。理想的多阶段系统可以：先用帧差法划定候选区域，在候选区域内用 Farneback 估计运动场，用 LK 角点做高精度验证。

### 6.4 整体指标偏低的深层原因

所有方法的绝对 IoU 均在 0.02~0.05 范围内，F1 在 0.04~0.08。这并非算法完全失败，而是由以下因素共同导致：

1. **真值掩膜极其稀疏**：运动像素占全图比例 <2%。即使算法检测到了大部分运动区域，FP 在绝对数量上仍然很大（IOU 分母大）
2. **两帧限制**：无法建立长时背景模型。帧差法仅用一帧做参考，光照的微小波动就能产生伪运动
3. **场景复杂度**：自动驾驶场景中，运动物体通常很小（远处车辆 <100 像素），且与背景纹理相似
4. **参数未逐样本调优**：所有 130 个样本使用相同参数，而最佳参数因光照、场景不同而异

### 6.5 失败案例分析

在所有样本中，IoU 最低的样本通常具有以下特征之一：
- **光照剧烈变化**（进出隧道、阴影区域）：帧差值远超阈值 25，导致大面积误检（FP 爆炸）
- **极小的运动物体**：远处行人或骑行者 <50 像素，低于形态学核的尺度，可能被开运算误删
- **相机抖动**：整帧的微小位移使所有边缘处产生帧差，形成环形误检带

### 6.6 算法局限性

1. **参数敏感性**：帧差阈值 25 在一种光照条件下最优，在另一种条件下可能完全失败。自适应阈值化（如基于局部直方图的动态阈值）是一个改进方向
2. **两帧场景退化**：仅有两帧相邻图像无法建立长时关联——连通域的跨帧匹配在两帧间没有验证机会；若不同样本 ID 恰好来自同一视频序列，可扩展为多帧关联跟踪
3. **校正 FOV 选择**：120° HFOV 是手工预设的。不同 FOV 会影响 Route B 的结果——更大 FOV 意味着更宽的覆盖但更严重的透视畸变
4. **未利用颜色信息**：所有运动提取在灰度域进行。颜色信息可以帮助区分阴影变化（主要为亮度变化）和真实运动（色度 + 亮度变化）

---

## 7. 结论

本实验围绕"鱼眼域直接处理 vs 校正后处理"的核心问题，实现并比较了三种传统运动提取算法。主要结论如下：

**（1）校正后处理（Route B）优于直接鱼眼域处理（Route A）**。在帧差法上，Route B 的全图 IoU 提升 11.6%，边缘区域提升 32.5%。校正确实缓解了鱼眼畸变对运动分析的不利影响，尤其在图像边缘区域效果显著。

**（2）鱼眼畸变对运动分析的负面影响是确定且可量化的**。中心区域的 F1 约为边缘的 1.7~1.9 倍。校正能缩小但不能消除这一差距，因为校正本身存在信息损失。

**（3）帧差法和 Farneback 光流各有所长、互为补充**。帧差法 Precision 高（0.040）但 Recall 不足（0.474）；Farneback Recall 极高（0.803）但 Precision 很低（0.025）。实际应用中应根据场景需求选择或结合使用。

**（4）传统方法在两帧场景下的绝对指标偏低**。所有方法的 IoU 均不超过 0.05，主要受真值稀疏性、两帧限制和固定参数的影响。多帧背景建模和自适应参数是自然的改进方向。
